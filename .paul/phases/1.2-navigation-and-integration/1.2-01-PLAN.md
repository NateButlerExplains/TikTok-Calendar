<objective>
## Goal
Wire up the calendar's navigation layer: sticky header with view switcher and prev/next arrows, error boundary for graceful failure handling, query param routing for deep links, Firebase Analytics instrumentation, and PWA configuration refinement.

## Purpose
Phase 1.1 built the rendering components (cards, views). Phase 1.2 makes them interactive and discoverable: users can switch views, navigate between days, bookmark specific dates, analytics tracks engagement, and the app installs on home screens. These are the connective tissue between components and user experience.

## Output
- `src/components/CalendarHeader.jsx` + `.module.css` — sticky nav with Day/3-Day toggle, prev/next arrows
- `src/components/BottomSheet.jsx` + `.module.css` — reusable modal for future use (month view)
- `src/components/ErrorBoundary.jsx` — catches React errors and displays fallback UI
- `src/App.jsx` (enhanced) — reads `?date=YYYY-MM-DD` query params, switches views, manages global state
- `src/firebase.js` (enhanced) — Firebase Analytics initialized and instrumented
- `vite.config.js` (updated) — PWA plugin configured with icons, descriptions
</objective>

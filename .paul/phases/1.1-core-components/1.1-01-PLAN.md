<objective>
## Goal
Build foundational React components and utilities for the Cyber Talks Calendar: global styling, event data model with test dataset, date/time utilities with timezone support, card components for guest/solo-talk/open-floor days, and both Daily and 3-Day view layouts. These form the core layer enabling calendar display and navigation.

## Purpose
Phase 1.1 delivers the rendering engine for all calendar views. Without these components and utilities, no view can display events. This phase is the prerequisite for Phase 1.2 (navigation/header) and 1.3 (polish/deploy).

## Output
- `src/styles/global.css` — dark theme, colors, typography, mobile breakpoints
- `src/styles/calendar.css` — layout utilities for calendar views
- `src/data/events.js` — populated with 7–10 days of sample events
- `src/hooks/useCalendarData.js` — date resolver, default fallback logic
- `src/utils/timeUtils.js` — timezone-aware time formatting and defaults
- `src/utils/icsGenerator.js` — .ics file generation with IANA timezone
- `src/components/GuestCard.jsx` — guest display with headshot, TikTok link
- `src/components/SoloTalkCard.jsx` — Nate's prepared talk display
- `src/components/OpenFloorCard.jsx` — open floor display
- `src/components/DayView.jsx` — single day, full detail, swipe, .ics, share
- `src/components/ThreeDayView.jsx` — 3-day layout (yesterday/today/tomorrow)
</objective>

  <action>
    Create src/utils/timeUtils.js exporting:
    
    resolveTime(date, event):
      - If event.time provided, return it
      - If date is Monday or Wednesday, return { hour: 12, minute: 0, durationMinutes: 60 }
      - All other days, return { hour: 20, minute: 0, durationMinutes: 60 }
      - Always use America/New_York timezone (handles EST/EDT)
    
    getStreamStart(dateString, event):
      - Parse dateString as YYYY-MM-DD
      - Resolve time from event or defaults
      - Return JavaScript Date object in America/New_York timezone (use date-fns-tz)
      - Example: "2026-04-21" + event → 8 PM EDT on that date
    
    formatTime(date, timeObj):
      - Format as "h:mm A Z" (e.g., "8:00 PM EDT", "12:00 PM EST")
      - Use date-fns format() with America/New_York timezone
    
    Use date-fns and date-fns-tz libraries (already in package.json)
  </action>

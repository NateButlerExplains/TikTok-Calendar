## Key Decisions Made

| Decision | Rationale | Impact |
|----------|-----------|--------|
| **Client-side .ics generation** | No server required for calendar export; ics library handles IANA timezone | Reduces backend complexity, improves UX (immediate download) |
| **useCalendarData hook** | Centralizes date/event resolution logic; handles missing dates gracefully | Clean separation of concerns; easy to test; reusable across views |
| **CSS Modules for components** | Avoids global namespace pollution; enables scoped styling per component | Maintainable, no style conflicts, mobile-friendly responsive rules |
| **Swipe via react-swipeable** | Mature library; handles touch + mouse gestures; minimal setup | Better mobile UX; works on desktop with mouse drag for testing |
| **Timezone: America/New_York** | All streams in Nate's timezone; EST/EDT handled automatically | No manual DST conversion needed; correct for all dates |

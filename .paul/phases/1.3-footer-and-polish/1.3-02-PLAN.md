## Goal
Test the Cyber Talks Calendar on mobile devices (iOS and Android) and desktop, verify responsive design, and audit performance. Document findings and create a performance report.

## Purpose
Phase 1.3-01 built the footer and legal pages. Phase 1.3-02 verifies the complete MVP works on actual devices with responsive design, fast load times, and no UI/UX bugs. This is a manual testing and verification phase—critical quality gate before final launch.

## Output
- Mobile testing report (iOS Safari, Chrome Android)
- Desktop testing report (DevTools 390px, Chrome desktop)
- Performance audit report (Lighthouse scores, bundle analysis)
- Bug fixes (if any critical issues found)
- Verification checklist (all ACs pass on all devices)
</objective>

<acceptance_criteria>

## AC-1: Desktop Responsive Design (DevTools 390px)
```gherkin
Given viewing the app at 390px width in Chrome DevTools
When testing all views (Day, 3-Day, legal pages)
Then layout is responsive and readable
And text is not cut off
And buttons are tap-friendly (min 44px height)
And no horizontal scrolling is needed
And footer doesn't overlap content
And all links are clickable
```

## AC-2: iOS Mobile Experience (iPhone Safari)
```gherkin
Given accessing the app on iPhone Safari (iOS 14+)
When navigating all views and legal pages
Then app loads without errors
And all gestures work (swipe navigation, taps)
And keyboard doesn't cover input fields
And status bar is visible
And Safe Area is respected (notch/home indicator)
And performance is acceptable (no jank)
```

## AC-3: Android Mobile Experience (Chrome Android)
```gherkin
Given accessing the app on Android Chrome
When navigating all views and legal pages
Then app loads without errors
And all touches register correctly
And navigation buttons are accessible
And footer is visible and scrollable
And no layout shifts occur on scroll
And performance is smooth (no lag)
```

## AC-4: Performance Audit
```gherkin
Given running Lighthouse audit on the app
When testing performance on mobile (slow 3G throttle)
Then Lighthouse score is 75+
And First Contentful Paint (FCP) is < 3s
And Largest Contentful Paint (LCP) is < 4s
And Cumulative Layout Shift (CLS) is < 0.1
And bundle size is < 600KB gzipped
And no console errors or warnings (except non-critical PWA warnings)
```

## AC-5: Deep Links & Browser History on Mobile
```gherkin
Given visiting the app with a deep link (?date=YYYY-MM-DD&view=three-day)
When on mobile device
Then the correct date and view load
And browser back/forward buttons work correctly
And URL persists when sharing links
```

## AC-6: PWA Installation on Mobile
```gherkin
Given accessing the app on iOS or Android
When installing to home screen
Then app is installable (prompt appears or add-to-home works)
And app icon and name display correctly
And app launches in standalone mode (no browser chrome)
And app works offline with cached assets
```

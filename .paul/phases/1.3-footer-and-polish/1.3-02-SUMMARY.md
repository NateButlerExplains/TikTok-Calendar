### Auto-fixed Issues

**1. ICS Download Import Error**
- **Found during:** Checkpoint 2 (iOS testing)
- **Issue:** TypeError: `(0 , Xs.default) is not a function`. Code imported ics as default export, but package only exports named functions.
- **Fix:** Changed `import ics from 'ics'` to `import { createEvent } from 'ics'`. Updated function call from `ics({...})` to `createEvent({...})`.
- **Files:** `src/utils/icsGenerator.js` (line 1, line 56)
- **Verification:** Desktop, iOS, and Android all successfully download .ics files
- **Commit:** `edf9e50` (fix: use named import for ics package)

**2. ICS Validation Error (Redundant Duration)**
- **Found during:** Checkpoint 2 (iOS testing)
- **Issue:** After fixing import, createEvent returned validation error. Cause: provided both `end` AND `duration` parameters. Package requires XOR (one or other, not both).
- **Fix:** Removed redundant `duration: { hours: 0, minutes: ... }` parameter. Kept `end` which was already calculated.
- **Files:** `src/utils/icsGenerator.js` (removed line with duration object)
- **Verification:** .ics download now works without validation errors on all devices
- **Commit:** `5425215` (fix: remove redundant duration parameter)

**3. PWA Build Issues (Deferred)**
- **Found during:** Checkpoint 1 (Desktop testing)
- **Issue:** Service worker and manifest files not being generated. Firebase serving HTML errors instead of PWA files. Manifest validation failed.
- **Decision:** Disable vite-plugin-pwa temporarily. PWA not critical for MVP (Phase 1.0). Planned for re-enable in Phase 2.
- **Impact:** Removed manifest/registerSW errors. App loads without PWA warnings.
- **Commits:** `11c27a3` (temp: disable PWA), `37d85ee` (reverted PWA rewrites)

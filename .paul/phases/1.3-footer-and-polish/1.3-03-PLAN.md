<acceptance_criteria>

## AC-1: Live Deployment Accessible
```gherkin
Given accessing https://cybertalks-guest.web.app from any location
When loading the production app
Then app loads without 404/500 errors
And page displays calendar views
And all assets load (CSS, JS, fonts, images)
And no console errors appear (non-critical PWA warnings acceptable)
```

## AC-2: All Features Work on Live Deployment
```gherkin
Given the live Cyber Talks Calendar is loaded
When testing Day View, 3-Day View, navigation, downloads, sharing, legal pages
Then all interactions work correctly
And .ics files download successfully
And share button copies URL to clipboard
And legal page links work (Privacy, Terms, About, Contact)
And back button on legal pages works
And deep links load correct date (?date=YYYY-MM-DD&view=X)
And no feature breaks on live that worked in testing
```

## AC-3: Performance Acceptable on Live
```gherkin
Given accessing the live app over typical mobile network (not localhost)
When navigating between views and downloading .ics files
Then First Contentful Paint (FCP) is < 3s
And Largest Contentful Paint (LCP) is < 4s
And no excessive layout shifts occur
And interactions feel responsive (no jank)
```

## AC-4: MVP Ready for Launch
```gherkin
Given Phase 1.3-01, 1.3-02, 1.3-03 all complete
When reviewing all acceptance criteria across all phases
Then all ACs pass
And no critical bugs remain
And app is stable and performant
And team/creator confirms readiness
Then MVP is approved for public launch
```

  <what-built>Complete Cyber Talks Calendar MVP deployed to production Firebase Hosting</what-built>
  <how-to-verify>
    1. Open https://cybertalks-guest.web.app in Chrome on desktop (clear cache: Cmd+Shift+R or Ctrl+Shift+R)
    2. Open DevTools console (F12) — check for JavaScript errors (PWA warnings OK)
    3. Test Day View:
       - Swipe or tap prev/next buttons
       - Verify responsive layout at desktop and mobile sizes
       - Tap .ics download button — verify file downloads
       - Tap share button — verify URL copied to clipboard
    4. Test 3-Day View:
       - Switch to 3-Day view
       - Verify layout shows yesterday/today/tomorrow
       - Test navigation arrows
    5. Test legal pages:
       - Click footer links: Privacy, Terms, About, Contact
       - Verify pages load without errors
       - Verify back button works on each page
    6. Test deep links:
       - Manually enter: https://cybertalks-guest.web.app?date=2026-05-15&view=three-day
       - Verify correct date and view load
    7. Performance check (informal):
       - Watch page load speed — should be fast (< 3s to interactive)
       - Scroll and navigate — no jank or layout shifts
    8. Document any issues found:
       - If issues: describe them for triage
       - If all working: note that verification passed
  </how-to-verify>

# Cyber Talks Calendar

## What This Is

A mobile-first web calendar for Cyber Talks TikTok live stream discovery. Viewers can see upcoming guest profiles, live stream times, download .ics calendar files for their calendar app, subscribe to email reminders, and share deep links. The app is hosted on Firebase with automatic GitHub Actions CI/CD deployments.

## Core Value

Help viewers discover when to tune in for Cyber Talks TikTok live streams and showcase guest profiles to drive TikTok follows.

## Requirements

### Core Features

- Daily View (landing, default) — full guest detail, swipe left/right, .ics download, share button
- 3-Day View — yesterday + today + tomorrow (today centered)
- Sticky view switcher (Day ↔ 3-Day)
- Prev/Next navigation arrows
- Deep link support (`?date=YYYY-MM-DD`)
- .ics download per day
- Share button (copy URL to clipboard)
- PWA install support (home screen icon)
- Dark/cyber theme (dark bg, cyan accent)
- Firebase Analytics (no UI; query/report only)

## Context

**Business Context:**
Daily TikTok live stream (Cyber Talks) with rotating guest speakers from cybersecurity space. Need to help viewers discover stream times and guest info. Creator: Nate Butler (@natebutlerexplains).

**Technical Context:**
Existing GitHub Actions CI/CD infrastructure. Firebase project (cybertalks-guest) configured with Hosting, Firestore (future), and Cloud Functions (future). React 18 + Vite for frontend bundling. Deploy pipeline auto-builds on push to main.

## Constraints

### Technical Constraints

- Single source of truth: `src/data/events.js` (edited locally, synced to Firestore in Phase 2)
- Timezone: America/New_York (handles EST/EDT automatically)
- Default times: Mon/Wed 12 PM EST, all others 8 PM EST
- .ics files must use IANA timezone format
- PWA manifest auto-generated by vite-plugin-pwa (no manual manifest.webmanifest in /public/)
- Firebase function rewrites have higher priority than SPA fallback in firebase.json

### Business Constraints

- Target launch: Within 48 hours (public immediately, no password/beta flag)
- Mobile testing: DevTools 390px (primary), iOS Safari, Chrome Android
- Guest headshots: ≤400x400px, <100KB pre-sized before commit
- Placeholder for unconfirmed guests: `/headshots/mystery.jpg`
- Guest workflow: Nate messages Claude → Claude edits events.js → push to main → auto-deploy within 2 min

# Cyber Talks Calendar

A mobile-first web calendar for **Cyber Talks** TikTok live streams. Viewers discover upcoming guest speakers, stream times, and TikTok profiles. Features swipe navigation, .ics calendar export, email reminders, and PWA installability.

**Live:** [cybertalkscalendar.web.app](https://cybertalkscalendar.web.app)

---

## Features

✨ **Calendar Views**
- Daily View — full guest detail, swipe navigation, .ics export
- 3-Day View — yesterday/today/tomorrow at a glance
- Responsive design — optimized for mobile (390px+)

👥 **Guest Profiles**
- Circular headshots with fallback
- Guest name, topic, TikTok profile link
- Solo-talk days with Nate's prepared topics
- Open-floor community discussion days

🔗 **Sharing & Integration**
- Per-day shareable links (`?date=YYYY-MM-DD`)
- Download .ics calendar files (Apple Calendar, Outlook, Google Calendar compatible)
- Email reminders (global or per-date subscriptions)

📱 **Mobile-First**
- Touch swipe gestures (left/right navigation)
- PWA installable (iOS home screen + Android)
- Offline support for cached dates

🌙 **Dark/Cyber Theme**
- Background: `#0a0a0f` (near-black)
- Accent: `#00f5ff` (electric cyan)
- Font: Inter (Google Fonts)
- Matches security/hacking aesthetic

## Tech Stack

| Layer | Technology | Purpose |
|-------|-----------|---------|
| **Frontend** | React 18 + Vite | Component architecture, fast bundling |
| **Styling** | CSS Modules + CSS Variables | Scoped styles, theme consistency |
| **Date Handling** | date-fns + date-fns-tz | Timezone-aware (America/New_York, EST/EDT) |
| **Gestures** | react-swipeable | Touch + mouse swipe navigation |
| **Calendar Export** | ics | .ics file generation (client-side) |
| **Backend** | Firebase (Hosting + Firestore + Cloud Functions) | Deployment, event sync, reminders, dynamic OG |
| **Analytics** | Firebase Analytics | User behavior tracking |
| **PWA** | vite-plugin-pwa | Manifest + service worker |

## Quick Start

### Prerequisites
- Node.js 18+
- npm or yarn
- Firebase CLI (for deployment)

### Installation

```bash
# Clone repo
git clone https://github.com/yourusername/tiktok-calendar.git
cd tiktok-calendar

# Install dependencies
npm install

# Set up Firebase
firebase login
firebase init hosting

# Start dev server
npm run dev
```

Visit `http://localhost:5173` — app opens on today's date in Daily View.

### Environment Variables

Create `.env.local`:

```env
VITE_FIREBASE_API_KEY=your_api_key
VITE_FIREBASE_AUTH_DOMAIN=your_project.firebaseapp.com
VITE_FIREBASE_PROJECT_ID=your_project_id
VITE_FIREBASE_STORAGE_BUCKET=your_project.appspot.com
VITE_FIREBASE_MESSAGING_SENDER_ID=your_sender_id
VITE_FIREBASE_APP_ID=your_app_id
```

## Data Model

**Edit `src/data/events.js`** to update the calendar. Nate + Claude collaborate locally; syncs to Firestore on deploy (Phase 2).

### Event Schema

```javascript
{
  date: "2026-04-21",                    // YYYY-MM-DD
  dayType: "guest",                      // "guest" | "solo-talk" | "open-floor"
  time: {                                // Optional; auto-defaults if omitted
    hour: 20,                            // 0–23 (20 = 8 PM)
    minute: 0,
    durationMinutes: 60
  },
  guests: [
    {
      name: "Alex Chen",
      headshot: "/headshots/alex-chen.jpg",
      tiktokUrl: "https://www.tiktok.com/@alexcyber",
      topic: "Zero Trust in 2026"
    }
  ]
}
```

### Default Times
- **Monday & Wednesday**: 12:00 PM EST
- **All other days**: 8:00 PM EST
- **Timezone**: America/New_York (automatically handles EST/EDT)

### Day Types
| Type | Shows | TikTok Link |
|------|-------|-------------|
| `guest` | Guest headshot + name + topic | ✅ Yes |
| `solo-talk` | Nate's headshot + prepared topic | ❌ No |
| `open-floor` | Nate's headshot + "Open Floor" | ❌ No |

Any date without an entry defaults to `open-floor`.

## Firebase Deployment

### Prerequisites
```bash
npm install -g firebase-tools
firebase login
firebase init hosting firestore functions
```

### Deploy

```bash
# Build and deploy
npm run build
firebase deploy

# Or deploy specific services
firebase deploy --only hosting          # Frontend only
firebase deploy --only firestore        # Firestore rules
firebase deploy --only functions        # Cloud Functions (Phase 2)
```

**Hosting URL**: `https://<project-id>.web.app`

### Firestore Setup (Phase 2)
```
collections/
├── events            # Read-only from client; synced from src/data/events.js
├── globalSubscribers # Viewers subscribed to all reminders
└── dateSubscribers   # Viewers subscribed to specific dates
```

Security rules block client writes to `events`; only Cloud Functions modify it.

## Architecture Decisions

| Decision | Rationale | Trade-off |
|----------|-----------|-----------|
| **Client-side .ics** | No server needed; instant download | Offline .ics generation only |
| **CSS Modules** | Scoped styles; no conflicts | Slightly larger output (negligible with Vite minification) |
| **America/New_York timezone** | Single global timezone; DST auto-handled | Assumes all streams in Nate's TZ |
| **events.js as source of truth** | Simple to edit; no admin portal to secure | Manual sync to Firestore (Phase 2) |
| **React + Vite over Next.js** | Fast bundling; simpler deployment to Firebase | No server-side rendering (static site) |
| **Swipe via react-swipeable** | Mature library; touch + mouse support | One extra dependency |

## Troubleshooting

### Swipe not working on desktop
- Ensure `trackMouse: true` in react-swipeable config (already set)
- Test with a click-and-drag motion

### Headshot image not loading
- Verify path in `events.js` (e.g., `/headshots/guest-name.jpg`)
- Check that image exists in `public/headshots/`
- Falls back to `/headshots/mystery.jpg` if missing (graceful degradation)

### Timezone showing incorrect time
- All times use `America/New_York` (GMT-5 EST / GMT-4 EDT)
- Verify system timezone is set correctly (for .ics import testing)
- Check `src/utils/timeUtils.js` if custom timezone needed

### Firebase deploy fails
```bash
firebase login
firebase use <project-id>
firebase deploy --debug
```

### Service worker issues
Clear cache: DevTools → Application → Clear storage → Unregister service workers
